Loaded into ``#connectors-modal-container`` on the registry list page by the "Ingest" button's ``hx-get`` to ``/ui/connectors/registry/ingest``. The shared app-shell modal controller (``modal-dialogs.js``) opens the inserted ```` via ``showModal()`` on ``htmx:afterSwap`` so it shows without an extra click and dismisses on close / ESC / backdrop (#1803). Two mutually-exclusive request shapes (the backend ``IngestRequest`` validator is the authority; the handler pre-checks to render a friendly inline error rather than a raw 422): * **catalog** -- a dropdown of ``/`` entries read from ``catalog_endpoint`` in-process. * **explicit quadruple** -- ``product`` / ``version`` / ``impl_id`` text inputs + one-or-more ``specs[].uri`` ``https://`` rows. Inline ``specs[].content`` byte upload is out of scope (CLI-only on-ramp). A **Dry-run** checkbox flips the submit between a sync parse-counts render (writes nothing) and the real async ingest (kicks the job + seeds the poll fragment). A **Write scope** select (#2209) rides both modes: ``global`` (the default, matching the REST/CLI omit-equals-global contract per #2085) or ``tenant`` (a tenant-curated ingest). The form posts only the discriminator -- the handler derives the tenant UUID from the authenticated operator, never from the client. Submit contract: the form ``hx-post``s to ``/ui/connectors/registry/ingest`` swapping ``#ingest-result`` (the in-modal result slot). A 422 friendly pre-check error, a catalog 422, the 503 ``LlmClientUnavailable`` panel, and the 202 job-poll seed all render into that slot so the operator stays in the modal. CSRF: the form carries the OWASP signed double-submit token on its OWN ``hx-headers`` (HTMX does not inherit the page ``hx-headers`` to an HTMX-injected fragment). The GET that rendered this modal minted the token + re-set the matching ``meho_csrf`` cookie. Context contract: * ``catalog_entries`` -- list[dict]: {ref, product, version, impl_id, ships_local} for the catalog dropdown (may be empty). ``ships_local`` (#1980) flags a row that ships a MEHO-authored spec/profile as package data (#1975 / #1976) -- a ``--catalog`` ingest of that row needs no operator-supplied spec upload. * ``csrf_token`` -- str: minted double-submit token for the POST. -#} {% from "_icons.html" import icon %}

Ingest a connector spec

Parse a vendor OpenAPI spec into the connector registry. Pick a catalog entry or supply an explicit product / version / impl_id with one or more https:// spec URLs. A real ingest runs as a background job you can watch to completion.
