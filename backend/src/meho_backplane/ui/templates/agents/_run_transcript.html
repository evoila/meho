Live-streaming model (mirrors broadcast/_feed.html) --------------------------------------------------- The HTMX ``sse`` extension owns the ``EventSource`` pointed at the bridge URL (``/ui/agents/{name}/run/stream?token=...``). The bridge proxies ``invoker.stream_events``, emitting one frame per event under a named ``event:`` (``turn`` / ``tool_call`` / ``tool_result`` / ``final`` / ``error``). One ``sse-swap`` per kind subscribes to each; the ``agentRunConsole`` Alpine controller hooks ``htmx:sse-before-message``, ``preventDefault()``s the raw swap (so a frame field containing markup is never parsed into live DOM -- the same injection class the broadcast surface closed in PR #1044), parses the JSON, and appends a typed transcript entry rendered via ``x-text`` (inert). SSE buffering #1389 is fixed + the bridge sets ``X-Accel-Buffering: no``, so frames flush per-event and the transcript paints turn-by-turn rather than only at completion. No Stop affordance: closing the page / navigating away closes the EventSource (the controller does not cancel the run). T9 #1833 adds the Stop button over the T8 #1828 cancel backend. XSS hardening: ``x-data`` is SINGLE-quoted and the token rides through ``| tojson`` (which escapes ``'`` to ``'`` and never emits a raw single quote), so a crafted token cannot break out of the attribute -- the same posture broadcast/_feed.html documents for its reflected ``op_id`` filter. -#} {% from "_icons.html" import icon %}
