{#- SPDX-License-Identifier: Apache-2.0 Copyright (c) 2026 evoila Group Curated-edge **bulk-import** panel (Initiative #1941 Task #1954). Rendered by ``GET /ui/topology/edges/bulk`` (panel-open) and re-rendered by ``POST /ui/topology/edges/bulk`` on a recoverable error (a parse failure, or the aggregated per-row ``invalid_bulk`` validation list). Flow: * Operator pastes an ``edges:`` doc into the textarea OR uploads a file. * **Dry-run preview** ``hx-post``s with ``dry_run=true`` and swaps the per-row create/update/conflict plan into ``#topology-bulk-result`` WITHOUT writing anything. * **Apply** ``hx-post``s with ``dry_run=false`` and swaps the applied summary in; the response carries ``HX-Trigger: meho:topology-edge-changed`` so an open graph view re-pulls. The two submit buttons share the same form; a hidden ``dry_run`` field is flipped by each button via ``hx-vals`` so the server branches on the flag (the REST shape: one route, a ``dry_run`` boolean). ``hx-disabled-elt`` disables both buttons during the in-flight request so a double-submit can't fire the apply twice. CSRF: the form carries the ``X-CSRF-Token`` header on its OWN ``hx-headers`` (HTMX does not inherit ``hx-headers`` to children). The panel mints + sets a fresh ``meho_csrf`` cookie on render so the POST carries a valid double-submit token. ``hx-encoding="multipart/form-data"`` lets the file ride along as a proper multipart part. The accepted file shape mirrors ``meho topology bulk-import``: a YAML/JSON doc with a top-level ``edges:`` list of ``{from, kind, to, note?, evidence_url?}`` rows, where ``from`` / ``to`` are a bare name string or a ``{name, kind}`` mapping. -#}
