{#- SPDX-License-Identifier: Apache-2.0 Copyright (c) 2026 evoila Group Async ingest job-poll fragment (Initiative #1839 Task #1886 T2). Modelled on ``agents/runs/_status_panel.html`` -- the "stop returning the polling element" idiom (https://htmx.org/docs/#polling). Rendered in two contexts: * Seeded into the ingest modal's ``#ingest-result`` slot by the POST submit handler when a real async ingest returned 202 + an ``IngestJobHandle`` (status ``running``). * Returned standalone by ``GET /ui/connectors/registry/ingest/jobs/ {job_id}`` on each poll tick. HTMX outerHTML-swaps it back into the ``#ingest-job-`` wrapper. Polling contract: while ``status == "running"`` the wrapper carries ``hx-get`` + ``hx-trigger="every Ns"`` + ``hx-swap="outerHTML"`` so HTMX re-fetches the panel on a timer. On ANY terminal status (succeeded / degraded / failed) -- AND on the ``job_lost`` 404 branch -- the wrapper DROPS those directives, so the poll cycle ends naturally. The job_lost branch is load-bearing: a process-local job evaporates on pod restart (``jobs.py:20``), so a 404 must render "job lost, re-check the registry list" and STOP -- never an infinite spinner. Context contract: * ``job_id`` -- str: the job UUID (drives the wrapper id + poll URL). * ``job`` -- dict | None: the projected job status (status / product / version / impl_id / error / error_class / ingestion / grouping); ``None`` on the job_lost branch. * ``job_lost`` -- bool: True when the in-process poll 404ed (process-local job gone); renders the terminal "re-check the registry list" panel, no poll. * ``poll_interval_seconds`` -- int: the self-poll cadence. -#} {% from "_icons.html" import icon %}
