Render branches (first match wins): 1. ``error_message`` set -> a plain error card. 2. ``hits`` non-empty -> one hit card each, with the fused_score + the per-signal RRF score/rank breakdown table. 3. ``searched`` true (a query ran, no error, no hits) -> "no matches". 4. otherwise (initial render, no query yet) -> a quiet prompt. Per-signal breakdown (the load-bearing render): Each hit carries ``bm25_score`` / ``bm25_rank`` and ``cosine_score`` / ``cosine_rank`` (float|None / int|None). A ``None`` rank means the document was absent from that signal's top-``candidate_limit`` candidate list -- it is rendered as an explicit "absent from this signal's top-N" marker, NOT a blank cell, so an operator can tell "scored 0" apart from "never appeared in that signal".
