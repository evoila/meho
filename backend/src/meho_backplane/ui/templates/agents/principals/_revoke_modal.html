Loaded into ``#principals-modal-container`` by a table row's "Revoke" button's ``hx-get`` to ``/ui/agents/principals/{name}/revoke``. The shared app-shell modal controller opens the inserted ```` via ``showModal()`` on ``htmx:afterSwap`` and dismisses on close / ESC / backdrop (#1803). Revoke is the Keycloak KILL SWITCH: it is **terminal** and destructive -- it disables the Keycloak client, which blocks all new token grants for this identity (tokens already minted stay valid until their ``exp``). There is no un-revoke path. Because a too-easy kill switch is a footgun (#1831 risk note), the dialog demands a **type-to-confirm of the principal name**: the destructive submit stays disabled until the operator types the name verbatim. The handler re-checks the typed value server-side, so a crafted POST cannot skip the confirm. The gate is an inline Alpine ``x-data`` (the chassis "zero inline script" CSP posture allows Alpine attributes; Alpine is vendored in base.html). ``x-model`` binds the input to ``typed``; ``:disabled`` keeps the submit inert until ``typed.trim() === expected``. Submit contract: the form ``hx-post``s to ``/ui/agents/principals/{name}/revoke``, swapping ``#principals-modal-container``. * Success -> 204 + ``HX-Redirect: /ui/agents/principals`` (the row swaps to its revoked state on reload). * 422 (typed-name mismatch) / 503 (Keycloak unconfigured) / 502 (Keycloak API failure) -> this dialog re-rendered with the banner. CSRF: the form carries its own ``hx-headers`` echo of the token; the route re-mints + re-sets the ``meho_csrf`` cookie on render (#1693). -#}
