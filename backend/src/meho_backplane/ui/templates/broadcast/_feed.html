Jinja's ``| tojson`` escapes ``<`` / ``>`` / ``&`` / ``'`` and U+2028/U+2029 but NOT the double-quote ``"`` -- it keeps the JSON string's own ``"`` delimiters and renders an inner ``"`` as a literal ``\"`` byte. Inside a DOUBLE-quoted ``x-data="..."`` those raw ``"`` bytes terminate the attribute early, letting a crafted ``op_id`` (e.g. ``" autofocus onfocus=alert(1) x="``) inject live event handlers -> reflected XSS. The ``x-data`` attribute is therefore SINGLE-quoted: ``tojson`` never emits a raw ``'`` (it escapes it to ``\u0027``), so single-quoted attribute cannot be broken out of by any scalar, and the ``"`` bytes ride harmlessly inside it. The static config has no raw ``'`` (``op_class_badge_json`` is ``json.dumps`` output -- only ``"``), so the single-quote delimiter is unambiguous.
