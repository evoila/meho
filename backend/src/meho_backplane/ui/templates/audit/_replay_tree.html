The forest is walked with Jinja's ``{% for ... recursive %}`` loop: ``{{ loop(node.children) }}`` re-enters the same body for each node's children, so the whole lineage renders without a fixed depth limit (``replay_session`` itself depth-bounds the closure). Each node reuses the T1/T2 row look -- the broadcast op_class badge palette (``badge_class``) -- and its "details" affordance ``hx-get``s the T2 drawer (``/ui/audit/show/{id}`` into ``#audit-drawer``), so the per-node aggregate-only 🔒 gate applies exactly as it does for a flat-query row: a ``credential_read`` / ``audit_query`` node in the tree never leaks its payload, the T2 drawer withholds it on open.
