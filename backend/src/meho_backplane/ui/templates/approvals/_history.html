{#- SPDX-License-Identifier: Apache-2.0 Copyright (c) 2026 evoila Group Approvals decision-history list partial (G10.8-T #1827). The HTMX swap target for the full-page console's status tabs, work_ref filter, and "Load more" offset pager. It is also included directly into ``index.html`` for the initial (server-rendered) page so both paths share identical markup. Status tabs / pager wiring: * Each status tab is an ``hx-get /ui/approvals/list?tab=`` that swaps ``#approvals-history`` in place. ``hx-include`` carries the live work_ref box so switching tabs preserves the filter. * The work_ref input fires the same fetch on a debounced ``keyup``. * ``hx-push-url`` keeps the browser URL in sync so a filtered view is bookmarkable / shareable. * "Load more" advances ``offset`` and APPENDS the next page (``hx-swap="beforeend"`` onto the rows list) rather than replacing the region, so paging never loses the rows already shown. * The whole region re-fetches the ACTIVE tab on the body-wide ``meho:approval-bump`` / ``meho:approval-decided`` events the app-shell bell dispatches, so a decision made elsewhere (or in this session's modal) drops out of the open Pending tab without a reload. No ``params`` / ``params_hash`` are ever in the projected row -- the render projection omits them by construction (approvals contract).
