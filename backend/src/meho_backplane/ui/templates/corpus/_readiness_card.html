{#- SPDX-License-Identifier: Apache-2.0 Copyright (c) 2026 evoila Group Docs Corpus collection readiness-card fragment (Initiative #1836 Task #1883). Surfaces a collection's lifecycle liveness: the ``status`` pill, ``doc_count``, ``last_ingested_at``, and the ``readiness.index_built`` managed-RAG footgun flag (reachable != answerable). Hosts the re-probe action. Rendered in two contexts (mirrors the connectors ``_fingerprint_card.html`` swap pattern): * Included by ``detail.html`` on the initial full-page render. * Returned standalone by ``POST /ui/corpus/collections/{key}/probe`` so a re-probe action swaps this fragment into ``#collection-readiness-card`` (``hx-swap="outerHTML"``) without a full-page reload. The re-probe button is rendered only when ``is_tenant_admin``; the server-side gate in the probe handler (``resolve_operator_or_403``) is the authority -- the template hides the affordance from operators who can't use it. Probe is slow (a managed-RAG rebuild serializes per project), so the button carries ``hx-disabled-elt="this"`` (disabled for the request duration) + ``hx-indicator`` driving an inline spinner so a naive synchronous probe does not look hung -- the same convention the connectors re-probe button uses. CSRF: the button declares its OWN ``hx-headers`` ``X-CSRF-Token`` -- HTMX does NOT propagate ``hx-headers`` to a swapped-in fragment from an ancestor (#1693), and the success swap replaces this card's markup, so the token must ride the element that issues the request. The route mints + re-sets the ``meho_csrf`` cookie on every card render so the double-submit pair lines up. Context contract (set by ``detail._render_readiness_card``): * ``collection`` -- dict: the projected full read shape (status / doc_count / last_ingested_at / readiness / ...). * ``is_tenant_admin`` -- bool: gates the re-probe button. * ``csrf_token`` -- str: minted double-submit token. -#}
